01

Compute

pick the runtime by control vs. ops

Decision tree

Containerized?
├─ NO
│  ├─ OS control / licensing / lift-shift ─→ Compute Engine
│  ├─ Event-driven snippet ───────────────→ Cloud Run functions
│  └─ Managed web app, no Docker ─────────→ App Engine Std
└─ YES
   ├─ Single stateless, scale-to-zero ────→ Cloud Run
   └─ Orchestration / many services
      ├─ Hands-off, pay per pod ──────────→ GKE Autopilot
      └─ Node-level control / DaemonSets ─→ GKE Standard

Trap: App Engine Flexible & Cloud Run with min-instances don't scale to zero. For "minimize cost when idle" → Standard / Cloud Run scale-to-zero.

Keyword → answer

  • "Lift-and-shift / existing license / OS control"Compute Engine
  • "Run code, scale to zero, single container"Cloud Run
  • "Microservices / Kubernetes / portability"GKE
  • "Reduce ops, pay per pod"GKE Autopilot
  • "Fault-tolerant batch, cheapest"Spot VMs
  • "Predictable steady 24/7, lowest cost"CUD
  • "BYOL per-core / physical isolation"Sole-tenant
  • "Recreate failed VMs automatically"MIG autohealing
  • "Survive a zone failure (VMs)"Regional MIG

03

Networking

VPC, hybrid, load balancing

VPC & connectivity

  • "Central team owns subnets for many projects"Shared VPC
  • "Connect 2 VPCs / orgs / merger"VPC Peering
  • "Private SaaS access, no IP overlap"PSC
  • "No-external-IP VM → Google APIs"Private Google Access
  • "Private VM outbound only"Cloud NAT

Hybrid link ladder

  • "Encrypted, low cost, quick" (99.99% = 2 tunnels)HA VPN
  • "Mid bandwidth, no colo"Partner Interconnect
  • "High bandwidth ≥10G, has colo, private"Dedicated Interconnect
  • "Only Google public APIs, no SLA"Direct/Carrier Peering

Trap: Interconnect is not encrypted by default → layer HA VPN / MACsec if encryption required.

Load balancer choice

  • "Global HTTP routing + CDN + WAF"Global Ext App LB
  • "Preserve client IP / UDP / non-TCP"Passthrough Net LB
  • "TLS offload, not HTTP, custom port"Proxy Net LB
  • "Between internal app tiers"Internal LB

Global = App LB & Proxy Net LB only; Passthrough is regional.

04

Security & IAM

least privilege, perimeters, keys

Service & access controls

  • "Prevent exfiltration even with stolen creds"VPC Service Controls
  • "Block SQLi/XSS/DDoS, rate limit, geo"Cloud Armor
  • "Access internal app w/o VPN, per-user"IAP / BeyondCorp
  • "Central misconfig / threat dashboard"Security Command Center
  • "Mask / redact / tokenize PII"Sensitive Data Protection
  • "Only signed/attested images deploy"Binary Authorization
  • "Owner/Editor/Viewer in prod"⚠ use predefined

Keys & identity

  • "Control key rotation, Google stores"CMEK (KMS)
  • "Google must never store the key"CSEK
  • "Keys fully outside Google"Cloud EKM
  • "FIPS 140-2 Level 3 hardware"Cloud HSM
  • "AWS/on-prem auth without keys"Workload Identity Fed.
  • "GKE pod → GCP perms, no node key"GKE Workload Identity
  • "Restrict creation to EU regions"Org Policy: locations

Hierarchy: Org → Folder → Project → Resource; policy is the union, inherited down. A child can't subtract a parent grant → use IAM Deny / Org Policy / VPC-SC.

05

Operations, Reliability & Cost

SRE, DR, billing

SRE & reliability

  • "Measured metric"SLI
  • "Internal target"SLO
  • "External contract w/ credits"SLA
  • "100% − SLO, freeze releases"Error budget
  • "Export logs long-term"Log sink → BQ/GCS

Disaster recovery

NeedStrategy
Cheapest, hrs OKBackup & restore
MinutesWarm standby
Near-zero, any costHot / active-active

RPO = data loss · RTO = downtime. Cloud SQL HA = synchronous standby in another zone.

Cost levers

  • "Steady 24/7 predictable"CUD (1/3 yr)
  • "Automatic, no commitment"Sustained Use
  • "Interruptible batch"Spot
  • "Analyze / chargeback spend"Billing → BigQuery

Budget alerts notify — they do not cap spend.
